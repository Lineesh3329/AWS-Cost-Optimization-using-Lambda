
# AWS-Cost-Optimization-using-Lambda

## Project Overview

- This project focuses on AWS Cost Optimization by automating the identification and deletion of unused EBS snapshots using AWS Lambda and Python (Boto3).
- Over time, unused snapshots can accumulate and increase storage costs.
- The Lambda function scans EBS snapshots, identifies snapshots associated with deleted or unused volumes, and automatically removes them.
- This helps reduce unnecessary AWS storage expenses, improve resource utilization, and maintain a cost-efficient cloud environment through automation.

## Project Execution Steps:

### 1. EC2 Instance Creation
1. AWS Console --> **EC2**
2.	Click Launch Instance
3.	Enter Instance Name:	**aws-cost-optimize**
4.	Choose AMI as **Ubuntu**
5.	Choose Instance Type as **t3.micro**
6.	Create Exsiting Keypair
7.	In Network	Allow **SSH (Port 22)**
8.	**EBS volume** - Default **8 GiB gp3** 
9.	Click Launch Instance

### 4. Lambda Function Creation
1.  Go to AWS → Lambda
2.	Click Create Function
3.  Choose Author From Scratch
4.	Enter Function Name	aws_cost_opt_ebs_vss
5.	Choose Runtime as **Python 3.x**
6.	Keep default Architecture **x86_64**
7.	Choose Execution Role	Default Role
8.	Click Create Function
